Для кого текст: для всех исследователей, которым нужно видеть картину того, что они читают, без фрустрации сложных программ

Цель воркшопа

Сделать 1-2 оформленных заметки по тексту, привязать их к одной из ваших исследовательских задач и тегировать для легкого поиска.

План воркшопа

Назначение системы

Хранение и организация знаний для их простого и «безотходного» использования в своих текстах и проектах — с помощью «Проводника», «Блокнота» и правил наименования файлов.

Чтобы вы:

  • знали, где (в каком проекте) можете использовать конкретную идею
  • знали, как именно ее использовать (в качестве чего)
  • знали, в чем суть идеи, не открывая файл с ней
  • могли найти нужную идею быстро по ключевым словам
  • видели группы идей прямо в папке, без сложных абстракций

Из чего состоит

  • Две папки:

    • Папка с простыми текстовыми файлами
    • Папка с источниками — PDF и сохраненными веб-страницами
    • *Папка со ссылками на простые текстовые файлы (создается автоматически — скриптом)

  • Два текстовых файла:

    • файл с проблемами, для решения которых вы собираете знания
    • файл для всех входящих идей — inbox.md

  • *Скрипт, обрабатывающий файл inbox.md и создающий отдельные текстовые файлы с каждой идеей/заметкой

  • Приложение для поиска файлов:

    • macOS: встроенный Spotlight или Alfred
    • Windows: встроенный поиск в «Проводник» или Everything
    • Linux: Albert или утилиты командной строки (fzf, ripgrep, etc.)

  • Приложение для редактирования текстовых файлов:

    • macOS: встроенный TextEdit, Typora, Bear, Visual Studio Code, Sublime Text, etc.
    • Windows: встроенный «Блокнот» или другой
    • Linux: встроенный vim, nano или другой

Файл с проблемами

Такой файл нужен, чтобы (1) оформить ваши исследовательские интересы и (2) обозначить их номера, чтобы присваивать их файлам.

Пример:

  • P1 — как возможен доступный интерфейс для углубления понимания в вашей доменной области?
  • P2 — почему у людей есть социальные институты, а у животных нет?
  • P3 — …

Файлы с заметками будут привязаны к номерам проблем, чтобы вы сразу понимали, где сможете использовать заметку. Например:

P1-МЕТОД.design.interaction_Браузер-позволяет-строить-интерфейсы-для-своих-нужд.md

В названии файла сразу же будет и проблема, и темы/теги (иерархические, если нужно). Более подробно про именование файлов — ниже.

Именование файлов

В названии файла рекомендуем указывать в таком порядке:

  1. (опционально) номер проблемы, для решения которой может пригодиться эта заметка
  • можно пропустить, если не подходит для решения проблем (но это сигнал, что вы, возможно, не решаете проблему, а собираете не очень нужную информацию)
  1. (опционально) функция заметки — в качестве чего её можно использовать, например:
  • МЕТОД — для секции о методологии
  • ТЕОРИЯ — для секции о вашем вкладе или рефлексии над исследованием
  • ФАКТ — для литературного обзора, это «ключевые результаты»
  • Прелесть в том, что вы можете придумать свои функции. Главное — использовать их одинаково.
  1. (обязательно) тему/теги, к которой относится заметка. Может быть вложенным, но не больше двух уровней:
  • хорошо: design.…, design.ux.… и design.visual.…
  • плохо: design.ux.usability.… (хотя вы можете так делать, если вам ОК длинные названия файлов). Вместо это лучше назвать просто usability.…
  1. (ОБЯЗАТЕЛЬНО) суть содержания заметки в виде предложения с подлежащим и сказуемым
  • рекомендуем придерживаться структуры: [кто/что] [делает/является] [что] [как/где]
  • рекомендуем использовать настолько короткие формулировки, насколько возможно без потери смысла названия для вас. Оптимально — чтобы все название файла помещалось в одну строку.

Теперь всё вместе: P1-МЕТОД.design.ixd_Браузер-позволяет-строить-интерфейсы-для-своих-нужд.md

P1-        МЕТОД.  design.ixd_     Браузер- позволяет-строить- интерфейсы- для-своих-нужд.md
[проблема][функция][область знаний][кто/что][делает/является]  [что]       [как/где]

Файл inbox.md

  • В него заносятся все идеи — свои или прочитанные/увиденные
  • В нем же происходит предобработка идей:
  • РЕКОМЕНДУЕМАЯ:
    • формулировка идеи своими словами в виде предложения с подлежащим и сказуемым
    • назначение темы, к которой идея принадлежит
    • переписывание идеи своими словами
  • ОПЦИОНАЛЬНАЯ и продвинутая:
    • обозначение типа отношения с другими заметками:
      • для чего является контекстом (seealso)
      • что опровергает (refutes)
      • что подтверждает (supports)
      • для чего служит примером (example)
      • что ставит под сомнение (questions)
      • И/ИЛИ другие пользовательские типы связи
    • обозначение типа — чем является заметка:
      • идея — предложение с подлежащим и сказуемым, содержащее одно или несколько понятий (основной тип)
      • понятие — одно понятие, как «Экономика замкнутого цикла»
      • инсайт — ваша собственная идея
      • факт — данные, статистика
      • организация
      • личность
      • событие
      • проблема/вопрос — тема вашего исследования
      • И/ИЛИ другие пользовательские типы записи

Алгоритм использования

Вся работа с информацией поделена на два этапа: - «захват» любых идей и сырой информации в файл inbox.md - редактирование идей в отдельных файлах - (опционально) визуализация отношений между сущностями в заметках

«Захват» идей в файл inbox.md

  • смотря/читая источник, заносите цитаты в файл inbox.md и указываете источник — URL-ссылку или название и тему/тег. Например:

Most of (text) editing is finding where you need to be https://youtu.be/ZH3iKbEiks0?t=828 Оптимизация навигации в разы ускоряет работу с текстом. P1.design.interaction

  • закончив читать источник (сразу или потом) смотрите inbox.md:
    • переписываете цитаты своими словами, чтобы получилось высказывание
    • формулируете суть идеи в виде предложения с подлежащим и сказуемым
    • обозначаете тему идеи или проверяете уже отмеченную — насколько она подходит
    • указываете связи между идеями
